# 🧩 Cozy Comfort – Service-Oriented System

Cozy Comfort is a Service-Oriented Architecture (SOA) based application designed to digitalize and optimize the supply chain process between manufacturers, distributors, and sellers. The system replaces traditional manual workflows with a scalable, modular, and API-driven solution.

## 🚀 Features

* 🔗 Service-Oriented Architecture (SOA) implementation
* 📦 Order Management System
* 🏪 Seller & Distributor Inventory Management
* 🏭 Manufacturer Production Tracking
* 🔔 Notification System for order updates
* 🔐 User Authentication & Session Handling
* 📊 Admin & User Interfaces (ASP.NET MVC)
* 🌐 RESTful API communication between services

## 💡 Project Overview

This system transforms a manual, error-prone supply chain into a modern digital platform by using independent services that communicate via APIs. Each business function (seller, distributor, manufacturer) is handled as a separate service, improving scalability, maintainability, and performance. 

## 🛠️ Tech Stack

* **Frontend:** ASP.NET Core MVC, HTML, CSS, Bootstrap
* **Backend:** .NET (C# Web API)
* **Database:** SQL Server / Entity Framework Core
* **Tools:** Visual Studio, Swagger, Postman, GitHub

## 🧠 Core Services

* 📦 **Order Management Service** – Handles order creation and tracking
* 🏪 **Seller Service** – Manages customer orders and requests
* 🚚 **Distributor Service** – Handles stock and fulfillment
* 🏭 **Manufacturer Service** – Provides production capacity & stock
* 🔄 **Fulfillment Service** – Coordinates order processing
* 🔔 **Notification Service** – Sends order updates

## 🏗️ System Architecture

* **Presentation Layer:** ASP.NET MVC Client Application
* **Service Layer:** Independent REST APIs (SOA)
* **Data Layer:** Centralized database

✔️ Modular design
✔️ Loose coupling
✔️ Independent service deployment

## 🧪 Testing

* ✅ Unit Testing (NUnit)
* 🔄 Integration Testing (Swagger & Postman)
* 👨‍💻 Functional Testing (End-to-end workflows)

## 🚀 Deployment

* Docker containerization for each service
* Kubernetes for orchestration and scaling
* CI/CD ready architecture

## 🎯 Objective

To build a scalable, maintainable, and flexible service-based system that supports real-time order processing, efficient inventory management, and seamless communication across multiple business entities.
